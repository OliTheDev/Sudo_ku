Description

A humorus take on the classic puzzle game displayed in terminal green.

How To Play

fill a 9x9 grid of cells with numbers 1-9, ensuring each number appears only once in every row, column, and 3x3 subgrid (or "box"). Each new game begins with different cells already filled in, these numbers cannot be changed. The game is finished when all the cells of the grid are filled.

1.Data

1.1 ID structure

Since the game is played on a 9x9 grid that means 81 cells. Each of these cells will be its own HTML element with a unique ID tag. If given just this ID tag we need to know three things about the cell that element represents.

  • what box it's in
  • what row it's in
  • what column it's in?

The protocol to encapsulate this cymantic information is as follows

Each ID must have a length of 3 characters(2 letters and 1 number). the first 2 characters will signify which box the cell is located within.
the first character can only be one of three letters a, b and c each representing the row of the containing box.
the second character can only be one of three letters d, e, and f each representing which column the containing box is in.
a visual example is given below as to how the grid is subdivided.

AD AE AF
BD BE BF
CD CE CF

the final character is a number representing the specific cell within that subgrid going left to right top to bottom.
an example

ad1

which represents the top most cell on the far left.
The AD subgrid would look like this

ad1 ad2 ad3
ad4 ad5 ad6
ad7 ad8 ad9
